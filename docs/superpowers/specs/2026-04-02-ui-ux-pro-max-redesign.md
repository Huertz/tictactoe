# UI/UX Pro Max Redesign — Tic-Tac-Toe
**Date:** 2026-04-02  
**Scope:** `templates/index.html` only (inline CSS + JS). Backend untouched.

---

## 1. Goals

Elevate the frontend to "UI UX Pro Max" quality: modern layout, clear state communication, animations/micro-interactions, full accessibility, and mobile-first responsiveness — while preserving all existing game functionality and the Flask backend API contract.

---

## 2. Hard Constraints

- Flask backend (`app.py`) is not modified.
- Routes `/`, `/move`, `/reset` — behavior and JSON contract unchanged.
- Minimax algorithm untouched; AI remains unbeatable.
- Board state remains server-side (Flask session, list of 9: `None | "X" | "O"`).
- Score counter stays client-side; resets on page refresh.

---

## 3. Design Direction: "Midnight Studio"

### 3.1 Color Tokens (CSS Custom Properties)

```css
--bg:          #0d0d14
--surface:     rgba(255, 255, 255, 0.04)
--border:      rgba(255, 255, 255, 0.08)
--accent-x:    #7c3aed   /* violet — X player */
--accent-o:    #06b6d4   /* cyan   — O player */
--text-primary: #f1f5f9
--text-muted:   #64748b
```

### 3.2 Typography

- Font: `Inter` via Google Fonts (weights 400, 600, 700).
- Fluid sizing via `clamp()` for headings and status text.

---

## 4. Layout

Single centered column, `max-width: 420px`, centered via flexbox on body.  
Order top-to-bottom:
1. Title
2. Mode toggle (segmented control)
3. Scoreboard (3 glass cards)
4. Turn indicator (player pill badges)
5. Board (3×3 grid)
6. New Game button

---

## 5. Board & Cell Design

- Cell size: `min(110px, 28vw)` — fluid, never overflows on small screens.
- Style: `border-radius: 16px`, `backdrop-filter: blur(12px)`, glass border.
- No text content in cells — replaced entirely by inline SVG.

### 5.1 SVG Stroke-Draw Animations

**X:** Two diagonal `<line>` elements. Each uses `stroke-dasharray` + `stroke-dashoffset` to animate the stroke drawing in. Duration: 250ms. Second line staggered 80ms. Stroke: `--accent-x`, `stroke-width: 5`, `stroke-linecap: round`.

**O:** Single `<circle>`. `stroke-dasharray` set to circumference; animates from `stroke-dashoffset: circumference → 0`. Duration: 300ms. Stroke: `--accent-o`, `stroke-width: 5`.

SVG elements are injected by JS into the cell on each move, replacing any prior content.

### 5.2 Hover Ghost

On empty cells: CSS `:hover` shows a faint (20% opacity) pseudo-element preview of the current player's symbol color as a ring/cross hint. Implemented with CSS only (no JS), driven by a `data-current-player` attribute on the board container updated by JS on each turn swap.

### 5.3 Cell Click Feedback

`transform: scale(0.93)` on `mousedown`, releasing to `scale(1)` — gives a tactile press feel.

### 5.4 Winning State

- Winning cells: colored `box-shadow` glow + gentle pulse keyframe.
- Non-winning cells: `opacity: 0.3` to frame the winner visually.

---

## 6. Turn Indicator

Replaces plain status text. Two pill badges side-by-side:
- Active badge: full color (violet for X, cyan for O) + glow ring + label "Your Turn".
- Inactive badge: muted/dimmed, no glow.
- Transition: CSS `background` + `box-shadow` smooth swap — no layout shift.
- In AI mode: O badge label reads "AI" instead of "Player O".
- While AI is computing: O badge shows a 3-dot pulse animation; board gets `pointer-events: none` to block double-clicks.

---

## 7. Result Modal

### 7.1 Trigger
On `winner` or `draw` in the `/move` response.

### 7.2 Appearance
- Frosted-glass card: `backdrop-filter: blur(20px)`, `background: rgba(13, 13, 20, 0.85)`.
- Slides up + fades in: `scale(0.85) → scale(1)` + `opacity: 0 → 1`, ~300ms cubic-bezier.
- Full-page backdrop overlay (semi-transparent, blurs the board behind it).

### 7.3 Content
- Large emoji: 🏆 (win) or 🤝 (draw).
- Result headline: e.g. "Player X Wins!" / "It's a Draw!".
- "Play Again" button — calls `/reset`, closes modal, resets turn indicator.
- Clicking backdrop also closes modal.

### 7.4 Confetti
On win only: 40-particle CSS confetti burst. Particles are `<div>` elements with randomized color, rotation, and fall distance — generated by JS and appended to the modal. No external library. Removed after animation completes.

### 7.5 Accessibility
- `role="dialog"`, `aria-modal="true"`, `aria-labelledby` pointing to headline.
- Focus trapped inside modal while open (Tab cycles through interactive elements).
- `Escape` key closes modal (equivalent to clicking backdrop).
- `aria-live="assertive"` region announces result to screen readers.

---

## 8. Scoreboard

Three glass cards in a row: X score | Draws | O score.  
On value increment: number animates with `scale(1.4) → scale(1)` pop (~200ms).  
Card labels update in AI mode: O card label becomes "AI".

---

## 9. Mode Toggle

Replaces two separate buttons with a single segmented control.  
- `role="radiogroup"` containing two `role="radio"` options.
- Animated background pill slides between options on selection.
- Switching mode calls `/reset` (existing behavior preserved).

---

## 10. New Game Button

- Full-width below board, violet fill.
- `hover`: slightly lighter fill.
- `active`: `scale(0.97)` press effect.

---

## 11. Accessibility

| Element | Implementation |
|---|---|
| Board cells | `role="button"`, `tabindex="0"`, `aria-label="Row N, Col N — empty/X/O"` |
| Mode toggle | `role="radiogroup"` + `role="radio"` + `aria-checked` |
| Result modal | `role="dialog"`, `aria-modal`, focus trap, `Escape` to close |
| Score updates | `aria-live="polite"` on scoreboard |
| Result announcement | `aria-live="assertive"` hidden element |
| Contrast | All text/interactive elements meet WCAG AA (white on violet/cyan buttons) |

---

## 12. Reduced Motion

All CSS transitions, keyframe animations, and JS-driven animations are wrapped in:
```css
@media (prefers-reduced-motion: reduce) { ... }
```
Under reduced motion: transitions are instant (`transition: none`), confetti does not fire, SVG draws appear instantly (dashoffset set directly, no animation).

---

## 13. Mobile Responsiveness

- `max-width: 420px` container; full-width padding on small screens.
- Board cells: `min(110px, 28vw)` — scales down on narrow viewports.
- Mode toggle and scoreboard: remain single-row but use `flex-wrap` as fallback.
- Touch targets: all interactive elements ≥ 44×44px (Apple HIG minimum).
- No horizontal scroll at any viewport width ≥ 320px.
